📱 MSG91 SMS + Firebase Setup

Step-by-step guide to activate SMS notifications for PackZen · ~45 minutes total

✅ What you get after this setup: Automatic SMS to customers when they book, when driver is assigned, when move starts, and when delivered. All for ~₹0.18 per SMS.
1
Set Up MSG91 Account (~10 min)
1
Create MSG91 Account

Go to msg91.com → Sign Up → enter your business details.

After signing up, add ₹100–200 credits to your wallet (minimum recharge). This gives you ~500–1000 SMS messages to start.

2
Register Sender ID

Go to SMS → Sender ID → Add New. Enter PKZNSM (6 letters, no spaces). Select "Transactional".

Approval takes 1–2 hours. This is what customers see as the sender name on their phone.

⚠️ DLT registration is mandatory in India since 2021. Without it, SMSes get blocked. See Step 4.
3
Get Your Auth Key

Go to API → Auth Key → copy the key. It looks like: 432198AaBbCcXx...

Keep this secret — never put it in your website code. You'll add it to Firebase only.

4
DLT Registration (Mandatory)

DLT (Distributed Ledger Technology) is required by TRAI for all business SMS in India.

Steps:

1. Go to Jio DLT or Airtel DLT and register as an entity (business).

2. Upload your GST certificate or business registration document.

3. Once approved (1–3 days), register your SMS templates. Use these exact texts:

// Template 1 — Booking Confirmed
Hi {#var#}, your PackZen booking {#var#} is confirmed for {#var#}!
Pickup: {#var#}. Est. cost: Rs.{#var#}. Track: packzen.in.
Queries: 9945095453

// Template 2 — Driver Assigned
Hi {#var#}, your PackZen driver {#var#} ({#var#}) is assigned
for booking {#var#}. Track live on packzen.in. Queries: 9945095453

// Template 3 — Move Started
Hi {#var#}, your goods are now in transit for booking {#var#}.
Track your driver live on packzen.in.

// Template 4 — Delivered
Hi {#var#}, your PackZen move {#var#} is complete! Please rate
your driver on packzen.in. Thank you for choosing PackZen!

// Template 5 — Damage Claim
Hi {#var#}, your damage claim (ID: {#var#}) for booking {#var#}
has been received. Our team will respond within 3 business days.
Queries: 9945095453

4. Copy the Template IDs from DLT and enter them in MSG91 → SMS Templates → link each template to its DLT ID.

💡 MSG91 has a DLT guide in their dashboard. Follow their "DLT Onboarding" wizard — it walks you through step by step.
2
Set Up Firebase CLI on Your Laptop (~10 min)
5
Install Node.js

Firebase CLI requires Node.js. Download from nodejs.org → choose the LTS version → install it.

After installing, open Terminal (Mac/Linux) or Command Prompt (Windows) and verify:

node --version   # should show v18 or higher
npm --version    # should show a number
6
Install Firebase CLI
npm install -g firebase-tools

Then login with your Google account (the one linked to Firebase):

firebase login   # opens browser, sign in with Google
7
Link to Your PackZen Project

Open Terminal in your GitHub repo folder (where index.html lives):

cd /path/to/your/packzen-repo
firebase use --add   # shows list of your Firebase projects
# Select your PackZen project from the list
# Give it alias: default
💡 Your Firebase project name is in your Firebase Console URL: console.firebase.google.com/project/YOUR-PROJECT-NAME
3
Deploy the Cloud Function (~10 min)
8
Install Function Dependencies
cd functions
npm install    # installs firebase-admin and firebase-functions
cd ..          # go back to repo root
9
Set Your MSG91 Key Securely

This stores your key on Firebase's servers — never in your code:

firebase functions:config:set \
  msg91.authkey="YOUR_MSG91_AUTH_KEY_HERE" \
  msg91.senderid="PKZNSM"

# Verify it was saved:
firebase functions:config:get
⚠️ Replace YOUR_MSG91_AUTH_KEY_HERE with your actual key from Step 3. Never commit this key to GitHub.
10
Deploy the Function
firebase deploy --only functions

# You should see something like:
# ✔  Deploy complete!
# Function URL: https://asia-south1-YOUR-PROJECT.cloudfunctions.net/sendSMS

Deployment takes 2–3 minutes. After this, every booking will automatically send an SMS.

11
Test It

Make a test booking on your site. Then check:

1. Firebase Console → Firestore → smsQueue — you should see a new doc with status: "sent"

2. Firebase Console → Functions → Logs — look for ✅ SMS sent to 91XXXXXXXXXX

3. Check the phone number you used — SMS should arrive within 30 seconds

✅ If status shows "sent" in Firestore, everything is working!
⚠️ If status shows "failed", check Functions logs for the error. Common issues: wrong auth key, DLT template not approved yet, or mobile number format issue.
12
Update Firestore Rules for smsQueue

Open your firestore.rules file and add this before the final deny-all rule:

match /smsQueue/{docId} {
  allow create: if isAuth();       // logged-in users can queue SMS
  allow read, update: if isAdmin();  // only admin can read queue
  allow delete: if false;
}

Then redeploy rules: firebase deploy --only firestore:rules
